## Session 16 (2026-04-22) — HEIC thumbnail support via libheif: 249 → 250 tests

### Completed Features (1 new test)
1. **HEIC thumbnail generation via libheif-rs** (Test 86): Added libheif-rs 2.7 crate
   for native HEIC/HEIF decoding. Thumbnail pipeline now handles HEIC files alongside
   JPEG/PNG/WebP. LibHeif decodes to RGB, then image crate handles resize/WebP encoding.
   Single-slot semaphore (Mutex) prevents OOM on concurrent large HEIC decodes.

### Implementation Details
- Added `libheif-rs = "2.7"` to workspace and tilde-photos dependencies
- `decode_heic()` function: HeifContext → primary_image_handle → LibHeif::decode → RGB plane → DynamicImage
- `open_image()` dispatcher: checks extension, routes HEIC to libheif, others to image crate
- `THUMBNAIL_LOCK` static Mutex: single-slot semaphore wrapping all thumbnail generation
- `generate_thumbnails_inner()` refactored out of `generate_thumbnails()` to avoid deadlock
  when `generate_video_thumbnail()` calls the inner function under the same lock
- 3 new unit tests: HEIC thumbnail, HEIC blurhash, HEIC extension detection
- Verified end-to-end: WebDAV PUT → watcher → organize → thumbnails (256.webp + 1920.webp)
