### Architecture Changes
- tilde-cal crate: Full CalDAV handler with auth, REPORT, sync-collection, getctag
- tilde-card crate: Full CardDAV handler with auth, REPORT, sync-collection
- Migration 003: calendars, calendar_objects, addressbooks, contacts, sync_changes tables
- CalDAV/CardDAV routers mounted at /caldav/ and /carddav/ with scope-based auth
- MCP expanded from 8 to 16 tools (added calendar, contacts, tasks, email tools)
- Default calendar and addressbook created on server startup

### Key Design Decisions
- Built CalDAV/CardDAV directly (not via RustiCal) since RustiCal integration was too complex
- Auth check done inside CalDAV/CardDAV handlers (not middleware) since nest_service doesn't support state-typed middleware
- Sync-collection uses monotonic sync_token per calendar/addressbook
- Soft-delete for calendar objects and contacts (deleted=1 flag) to support sync-collection 404 responses
