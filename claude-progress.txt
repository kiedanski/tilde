   temp file via BufWriter instead of buffering full body in memory. Includes fsync

   password prompting, TLS mode selection, backup keypair generation, recovery code
   display. Non-interactive when env vars set. Does NOT auto-start systemd.

   recovery code (groups of 4), stores hash in kv_meta, prints to terminal once.

6. **Export tar.zst** (Test 158): `tilde export run --format tar.zst` compresses
   the export directory into a zstandard-compressed tar archive.

### Completed Features (6 new tests)

   job processor runs every 5s to process pending thumbnail jobs.

2. **systemd unit file generation** (Test 180): `tilde install` command writes hardened
   systemd unit file to /etc/systemd/system/tilde.service with Type=notify, WatchdogSec=30s,
   full security hardening stanza. Creates system user 'tilde'. Idempotent.

3. **Cross-reference URI stability** (Test 201): Export now includes manifest.json with
   UUID→path mappings for photos, and links.json with all tilde:// URIs. Import reads
   links.json and inserts cross-references into DB. UUIDs stable across export/import.

4. **Self-hosted update manifest mirror** (Test 222): Added UpdatesConfig to config with
   manifest_mirror field. `tilde update check` uses mirror URL when configured.

5. **CLI progress indicators** (Test 250): `tilde photos reindex` shows file count progress.
   `tilde export` shows per-section progress with timing summary.

6. **CLI confirmation prompts** (Test 251): Destructive operations (collection delete) now
   prompt for confirmation. `--yes`/`-y` flag skips prompts for scripting.
