   - PROPFIND now includes custom properties in responses

3. **Nextcloud Chunked Upload v2 (Tests 42, 43, 44)**:
   - MKCOL /dav/uploads/<user>/<session-id>/ creates upload session
   - PUT chunks to session directory
   - MOVE finalizes: assembles chunks in order, atomic rename to destination

   - Per-session tracking in chunked_uploads table (bytes_received, chunk_count)
   - 24-hour session expiry with expires_at timestamp
   - Startup cleanup of expired sessions and staging directories
   - Disk space pre-check via fs2 when OC-Total-Length header present
   - 507 Insufficient Storage response when disk is full
   - Separate uploads router mounted at /dav/uploads/

## Session 3 (2025-04-22) — Auth, Rate Limiting, WebDAV

### Completed
1. **Session tokens with sliding TTL (Test 9)**:
   - Fixed token format: 46 chars (14 prefix + 32 hex from 16 random bytes)
   - Added sliding TTL: expires_at extends from current time on each valid use
   - Added POST /api/auth/login endpoint (JSON body: {password, user_agent?, source_ip?})
   - Added auth middleware for Bearer tokens (session + MCP) and Basic auth (app passwords)
   - Added GET /api/auth/verify and GET /api/auth/session endpoints

2. **Rate limiting on auth endpoints (Test 11)**:
   - Per-IP rate limiting on login: 5 attempts per 15 minutes
   - 429 Too Many Requests with Retry-After header
   - Valid login clears rate limit for that IP
   - In-memory HashMap<String, RateLimitEntry> in AppState

3. **Password reset (Test 14)** and **Session list/revoke (Test 15)**:
   - Already implemented in session 2, verified end-to-end
   - Updated session list to show user_agent and source_ip columns

4. **XDG and systemd path resolution (Tests 3-4)**:
   - Fixed XDG_CONFIG_HOME, XDG_DATA_HOME, XDG_CACHE_HOME support
   - Added systemd mode: STATE_DIRECTORY → /var/lib/tilde, /var/cache/tilde, /etc/tilde
   - XDG env vars now checked before `directories` crate fallback

5. **Metrics localhost-only (Test 19)**:
   - Added ConnectInfo<SocketAddr> extractor
   - Returns 403 for non-loopback IPs

6. **Host header allowlist (Test 24)**:
   - Middleware checks Host header against server.hostname
   - localhost and 127.0.0.1 always allowed
   - Returns 403 for unknown hosts

7. **WebDAV Class 1 (Tests 29-38)**:
   - Full tilde-dav crate implementation
   - OPTIONS: DAV: 1 header, method list
   - GET/HEAD: file download with Content-Type and ETag
   - PUT: atomic write (tmp → rename), ETag generation, SHA-256, files table upsert
   - PUT with If-Match: 412 Precondition Failed on ETag mismatch
   - DELETE: file and recursive collection deletion
   - MKCOL: directory creation, 405 if exists, 409 if missing parent
   - MOVE: preserves oc:id (UUID), recursive child path update
   - COPY: new UUID for destination, content duplication
   - PROPFIND Depth:0 and Depth:1: XML multistatus response with DAV and OC properties
   - PROPPATCH: minimal stub (acknowledges request)
   - LOCK: 200 OK stub (macOS Finder compatibility)
   - Mounted at /dav/files/* via nest_service

8. **Config env var fix**:
   - Fixed figment env var mapping to lowercase before matching
   - TILDE_SERVER__LISTEN_PORT now works (double underscore = nesting separator)
   - Added dev port override: TILDE_SERVER__LISTEN_PORT=8080 in .env

### Architecture Changes
- AppState.db changed from Mutex<Connection> to Arc<Mutex<Connection>> (shared with DavState)
- build_router() now takes both SharedState and SharedDavState
- ConnectInfo<SocketAddr> required — server uses into_make_service_with_connect_info()
- base64 crate added for Basic auth decoding
